# Playbook - Refatorar Sem Quebrar

## Objetivo

Melhorar design preservando comportamento.

## Procedimento

1. Identificar comportamento atual.
2. Criar teste de proteção.
3. Fazer menor refactor.
4. Rodar testes.
5. Revisar diff.
6. Evitar cleanup lateral.
