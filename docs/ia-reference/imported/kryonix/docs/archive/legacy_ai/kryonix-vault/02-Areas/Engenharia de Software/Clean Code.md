# Clean Code

## Objetivo

Escrever código legível, testável e barato de manter.

## Princípios úteis

- nomes específicos;
- funções coesas;
- baixo acoplamento;
- erros explícitos;
- validação nas bordas;
- evitar abstração prematura;
- testes para comportamento.

## Sinal de qualidade

Você consegue explicar o código, testar comportamento e alterar uma regra sem quebrar áreas não relacionadas.
