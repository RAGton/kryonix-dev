# Playbook - Debug de Producao

## Objetivo

Resolver incidente sem piorar o sistema.

## Procedimento

1. Definir impacto.
2. Congelar mudanças arriscadas.
3. Coletar logs e métricas.
4. Formular hipóteses.
5. Testar hipótese menos destrutiva.
6. Mitigar.
7. Corrigir.
8. Registrar pós-incidente.
