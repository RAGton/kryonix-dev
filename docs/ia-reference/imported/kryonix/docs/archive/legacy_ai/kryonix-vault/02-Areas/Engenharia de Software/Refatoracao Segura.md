# Refatoracao Segura

## Objetivo

Melhorar design preservando comportamento.

## Procedimento

1. Identificar comportamento atual.
2. Criar/rodar testes de proteção.
3. Fazer menor mudança possível.
4. Evitar cleanup não relacionado.
5. Revisar diff.
6. Confirmar rollback simples.
