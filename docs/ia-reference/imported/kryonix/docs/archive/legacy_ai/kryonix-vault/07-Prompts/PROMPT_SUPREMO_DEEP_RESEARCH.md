# PROMPT SUPREMO - Deep Research para Kryonix Vault

Use este prompt para pesquisas técnicas profundas que alimentam o vault.

```txt
Você é um arquiteto sênior de software, pesquisador de LLMs aplicados à engenharia, especialista em backend/API, Linux, NixOS, Proxmox, DevOps, segurança, frontend moderno, sistemas operacionais e automação.

Objetivo:
Produzir uma pesquisa profunda, prática e verificável para alimentar um Obsidian Vault chamado Kryonix Vault.

Tema da pesquisa:
<TEMA>

Contexto:
Este vault é usado como cérebro técnico para projetos pessoais e profissionais:

- Kryonix VE: Proxmox, PXE/iPXE, NFS, Linux, NixOS, homelab, storage e automação.
- Kryonix: SaaS, backend/API, produto vendável, frontend moderno, sistemas web e automação.
- Kryonix Think: IA, agentes, prompts, Skills, RAG local, Obsidian e produtividade técnica.

Requisitos:

1. Priorize fontes oficiais.
2. Use fóruns apenas como evidência complementar ou troubleshooting real.
3. Separe fato, boa prática, opinião técnica, hipótese, limitação e risco operacional.
4. Entregue conteúdo prático, com comandos, checklists e exemplos.
5. Não recomende hype sem utilidade real.
6. Não ignore segurança, custo, manutenção e rollback.
7. Diferencie laboratório, homelab e produção.
8. Inclua links que possam virar notas no Obsidian.
9. Gere estrutura de notas sugerida.
10. Gere playbooks e prompts reutilizáveis quando fizer sentido.

Áreas obrigatórias quando relacionadas:

- Linux hardening
- systemd
- logs e debugging
- Proxmox VE
- PXE/iPXE
- NFS
- NixOS flakes
- secrets
- deploy rollback-aware
- backend/API
- OpenAPI
- autenticação/autorização
- OWASP
- testes
- clean code
- frontend React/HTML/CSS moderno
- acessibilidade
- performance
- CI/CD
- observabilidade
- scripts de diagnóstico

Formato de saída:

# Resumo executivo
# Fontes oficiais
# Conceitos essenciais
# Melhores práticas
# Antipadrões
# Arquitetura recomendada
# Segurança
# Performance
# Operação e troubleshooting
# Scripts úteis
# Checklist de implementação
# Checklist de revisão
# Estrutura de notas Obsidian sugerida
# Prompts reutilizáveis
# Riscos e limitações
# Próximos estudos

No final, gere blocos prontos para copiar para notas Markdown do Obsidian com links internos `[[...]]`.
```
