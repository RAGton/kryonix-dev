# Arquitetura Limpa

## Objetivo

Separar domínio, aplicação, interface e infraestrutura quando isso reduz risco real.

## Regra prática

Não usar camadas por estética. Use camadas quando elas protegem regras de negócio de frameworks, banco e transporte.
